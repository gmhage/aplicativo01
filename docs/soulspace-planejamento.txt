# PRD — Soulspace

## O Produto
Soulspace é um app desenvolvido para resolver o seguinte problema:
💔 O PROBLEMA CENTRAL:
"Solteiros(as) estão emocionalmente PREPARADOS para ter um relacionamento FELIZ, mas não sabem COMEçar e nem CONSERvar."

Desdobramento da dor:
Situação	O que acontece HOJE (jeito errado)	Impacto emocional
1. Solteiro quer namorar MAS...	❌ Entra no Tinder/Bumble → encontra pessoas superficiais, só quer "ficar" → frustração 
Frustração, desconfiança, baixa autoestima
2. Solteiro tem ANSIEDADE SOCIAL	❌ Evita encontrar pessoas novas → fica isolado → solidão cresce 
Solidão, depressão, autoisolamento
3. Solteiro NAMORA MAS...	❌ Repete erros do passado (escolhe pessoa errada, não comunica, não desenvolve) → relacionamento falha 
Ciclo de namoros que falham, desesperança
4. Solteiro quer "AMAR-SE PRIMEIRO"	❌ůže entender o que quer, não sabe suas "gatilhos" emocionais → não evolui internamente 
Não cresce, não prepara para receber alguém
⚠️ O QUE AS PESSOAS FAZEM DE JEITO ERRADO, LENTO OU COMPLICADO:
1. TENTAM RESOLVER NO TINDER/BUMBLE (jeito errado)
Erro	Por que é errado?
Foco em "encontrar alguém"	Não "preparar-se interior" → encontram pessoa, mas não sabem desenvolver relacionamento
Superficialidade	Fotos + 3 linhas de texto → não revela personalidade, valores, intenção real
Gamificação	"Swipe" como jogo → pessoas tratam como consumo, não como conexão humana
Resultado:	6 reinstalções do app por ano 
 → frustração constante
2. TENTAM TERAPIA ONLINE (jeito lento)
Erro	Por que é lento?
Custo alto	R$ 150–300/sessão → 1x/semana = R$ 600–1.200/mês → inacessível para maioria
Acesso limitado	Só 1x/semana → dor emocional袭来 → não tem suporte imediato
Foco em "curar"	Terapia tradicional = tratar doença → stigmatiza, solteiro não quer "doente"
Resultado:	3–6 meses para ver evolução → solteiro desiste no meio
3. TENTAM MEDITAÇÃO GENÉRICA (jeito complicado)
Erro	Por que é complicado?
Apps genéricos (Calm, Headspace)	Meditação para "todos" → não específica para SOLTEIRO + ansiedade social + relacionamento
Não conecta com objetivo	Medita para "relaxar" → não para "preparar para relacionamento"
Pouca personalização	Não entende seu histórico emocional → meditação não resolve sua dor específica
Resultado:	Medita 2 semanas → não vê evolução → desiste
4. TENTAM AUTOCONHECIMENTO SOZINHO (jeito errado)
Erro	Por que é errado?
** leituras aleatórias**	Livro de autoajuda → não tem estrutura, não sabe aplicar no dia a dia
Diário na mão/papel	Esquece, não analisa padrões → não vê evolução emocional
Nenhuma orientação	"O que eu sinto?" → não sabe nomear, não sabe resolver
Resultado:	6 meses tentando → não evolui, não entende padrões
5. TENTAM GRUPOS DE SOLTEIROS (jeito complicado)
Erro	Por que é complicado?
Eventos presenciais	Requer sair de casa, enfrentar ansiedade social → difícil para quem tem fobia
Grupos Facebook/WhatsApp	Sem filtro, sem moderação → encontra pessoas falsas, sem intenção séria
Não tem estrutura	"Bate-papo" → não tem exercício, não tem evolução mensurável
Resultado:	Encontra 2–3 pessoas → nenhuma séria → frustração

## Tratamento de erros
🔴 1. INTERNET CAI (falha de conexão)
Situação	O que o app faz	Mensagem ao usuário
Internet cai durante uso	App detecta automaticamente → mostra banner amarelo no topo	"Você está sem internet. Seus dados serão salvos e enviados quando voltar."
Usuário tenta salvar diário	Salva localmente (no celular) → não perde dado	"Diário salvo no seu celular. Enviaremos quando internet voltar."
Usuário tenta abrir IA Coach	Mostra tela de erro → não carrega chat	"IA Coach precisa de internet. Volte quando conexão voltar."
Internet volta	App sincroniza automaticamente → dados salvos localmente vão ao Firebase	"Internet voltou! Dados sincronizados." (banner verde)
🔴 2. DADO INVÁLIDO (campo preenchido errado)
Situação	O que o app faz	Mensagem ao usuário
Usuário digita nome vazio	Campo fica vermelho → botão "Continuar" não ativa	"Por favor, digite seu nome." (texto vermelho abaixo do campo)
Usuário seleciona idade < 18	Dropdown não permite → mostra erro	"Idade mínima: 18 anos."
Usuário digita email sem "@"	Campo fica vermelho → mostra ícone de erro	"Email inválido. Use exemplo@exemplo.com"
Usuário tenta login com email errado	Erro no Firebase → mostra popup	"Email ou senha incorretos. Tente novamente."
🔴 3. PAGAMENTO RECUSADO (Stripe falha)
Situação	O que o app faz	Mensagem ao usuário
Cartão recusado (saldo insuficiente)	Stripe retorna erro → mostra popup vermelho	"Pagamento recusado. Verifique saldo do cartão ou tente outro cartão."
Cartão inválido (número errado)	Stripe valida → mostra erro no campo	"Número do cartão inválido. Verifique os 16 dígitos."
CVV errado	Stripe valida → mostra erro no campo	"CVV inválido. Use 3 dígitos (lado do cartão)."
Cartão expirado	Stripe valida → mostra erro no campo	"Cartão expirado. Use cartão atual."
Usuário tenta outro cartão	Campo cartão limpa → usuário digita novo	"Tente outro cartão."
Pagamento sucesso	Stripe confirma → mostra popup verde	"Pagamento processado! Bem-vinda ao SoulSpace 🎉"
🔴 4. IA NÃO RESPONDE (OpenAI falha)
Situação	O que o app faz	Mensagem ao usuário
OpenAI timeout (5s sem resposta)	Chat mostra mensagem de erro	"IA demorando para responder. Tente novamente."
OpenAI erro (500 Internal)	Chat não carrega → mostra popup	"IA temporariamente indisponível. Voltaremos em alguns minutos."
Usuário reenvia mensagem	App tenta novamente → 3x de retry automático	(se sucesso) "IA respondeu!"
3 retries falharem	Mostra opção "Contato humano"	"IA não está respondendo. Fale com nosso suporte: [email]"
🔴 5. LOGIN FALHA (Firebase erro)
Situação	O que o app faz	Mensagem ao usuário
Email não existe	Firebase retorna erro → mostra popup	"Email não encontrado. Crie uma conta."
Senha incorreta	Firebase valida → mostra erro no campo	"Senha incorreta. Tente novamente."
Google/Apple login falha	Popup erro → usuário seleciona outro método	"Login Google falhou. Tente com e-mail."
Session expirada	App força logout → volta tela login	"Sessão expirada. Faça login novamente."
🔴 6. DIÁRIO NÃO SALVA (banco de dados erro)
Situação	O que o app faz	Mensagem ao usuário
Firebase erro (500)	Salva localmente (celular) → não perde dado	"Salvo no seu celular. Enviaremos quando Firebase voltar."
Campo de texto vazio	Botão "Salvar" não ativa → mostra erro	"Por favor, digite algo no diário."
Texto muito longo (>10.000 chars)	Limite Firebase → mostra erro	"Texto muito longo. Use até 10.000 caracteres."
Salva sucesso	Firebase confirma → mostra popup verde	"Diário salvo! IA analisa seus padrões."
🔴 7. NOTIFICAÇÃO PUSH NÃO ENTRA (OneSignal erro)
Situação	O que o app faz	Mensagem ao usuário
OneSignal falha	App não mostra erro → só não entrega notificação	(silencioso)
Usuário desativou notificações	App detecta → mostra alerta no primeira uso	"Notificações desativadas. Ative para receber lembretes."
Usuário ativa notificações	OneSignal atualiza → notificação funciona	"Notificações ativadas! Você receberá lembretes."
🔴 8. APP CRASHA (erro crítico do sistema)
Situação	O que o app faz	Mensagem ao usuário
App fecha inesperadamente	Sentry captura erro → envia relatório	(silencioso)
App abre de novo	Sentry detecta crash anterior → mostra popup	"App travou antes. Corrigimos o erro. Continue de onde estava."
Usuário perde dados	Firebase restaura last save → dados voltam	"Dados restaurados! Você estava no diário de hoje."
📊 TABELA COMPLETA DE TRATAMENTO DE ERROS:
Erro	Detectado por	Ação do app	Mensagem ao usuário
Internet cai	App detecta automaticamente	Salva localmente → sincroniza quando volta	"Sem internet. Dados salvos no celular."
Dado inválido	Validação no campo	Campo fica vermelho → botão não ativa	"Campo inválido. Corrija."
Pagamento recusado	Stripe retorna erro	Mostra popup → campo limpa	"Pagamento recusado. Tente outro cartão."
IA não responde	OpenAI timeout/erro	3 retries automáticos → opção suporte	"IA indisponível. Suporte: [email]"
Login falha	Firebase erro	Mostra popup → outra opção login	"Email/senha incorretos."
Diário não salva	Firebase erro	Salva localmente → sincroniza	"Salvo no celular. Enviaremos quando voltar."
Push não entra	OneSignal falha	Silencioso (não mostra erro)	(nenhuma)
App crasha	Sentry captura	Restaura dados → mostra popup	"App travou. Corrigimos. Continue."
✅ PRINCÍPIOS DE TRATAMENTO DE ERROS (profissional):
1. Não perca dado do usuário:
Situação	Princípio
Internet cai	Salva localmente → não perde diário
Firebase erro	Salva localmente → sincroniza quando volta
App crasha	Restaura last save → não perde progresso
2. Mensagem clara e útil:
Mensagem ruim	Mensagem boa
"Erro 500"	"IA temporariamente indisponível. Voltaremos em alguns minutos."
"Pagamento falhou"	"Pagamento recusado. Verifique saldo ou tente outro cartão."
"Campo inválido"	"Por favor, digite seu nome."
3. Retry automático (não exige ação do usuário):
Erro	Retry
IA não responde	3x automáticos → se falhar, opção suporte
Internet volta	Sincroniza automaticamente
Firebase erro	Tenta salvar novamente 2x
4. Opção de suporte humano:
Situação	Suporte
IA falha 3x	"Fale com suporte: [email]"
Pagamento falha 3x	"Contate suporte: [email]"
App crasha	"Reporte erro: [email]"
🎯 FRASE-CHAVE PARA TRATAMENTO DE ERROS:
"SoulSpace não perde dado do usuário: (1) salva localmente quando internet cai, (2) sincroniza automaticamente quando volta, (3) restaura last save quando app crasha. Mensagem sempre clara e útil, retry automático, opção de suporte humano."
